# 自訂分類 — 討論與實作計畫

## 先釐清：自訂分類 vs 官方分類

Shopee 有兩種分類系統：

| | 官方分類 | 自訂分類 |
|---|---|---|
| 用途 | 蝦皮平台搜尋/瀏覽用 | 賣場內部整理用 |
| 設定位置 | 新增商品頁面選擇 | 賣場佈置 → 我的賣場分類 |
| 大量上傳 | Excel `ps_category` 欄位 | **不支援** |
| 跨店通用 | 同一組 ID | 每間店 ID 不同 |
| 設定時機 | 上架時 | 上架後 |

**結論：自訂分類無法在上架時設定，只能上架後補設定。**

## 自訂分類的操作方式

在「賣場佈置 → 我的賣場分類」→ 選一個分類 → 「查看詳情」→ 「新增商品」→ 搜尋商品名稱 → 勾選 → 新增。

這意味著：**只要商品標題有關鍵字，就可以批次搜尋出來一次加入。**

## 你的上架系統架構

```
product-catalog-tw.json
  → batch-upload.js (Chrome Extension)
    → 開 seller.shopee.tw/portal/product/new
      → fillProductData (填寫商品資料)
      → uploadMedia (上傳圖片)
      → 點擊「儲存並上架」
```

上架管線本身不處理自訂分類，需要一個**後處理步驟**。

## 最優解：標題關鍵字策略 + 上架後批次指派

### Phase 1 — 統一分類命名

JSON 已有的 `category` 欄位（非 `ps_` 前綴，僅供內部用）要跟賣場自訂分類名稱對齊：

目前的 JSON category 值：
```
實用工具: 18 筆
語音/配音/變聲: 15 筆
圖片生成/編輯: 14 筆
音樂生成: 12 筆
影片生成: 10 筆
影片處理: 9 筆
虛擬換衣/換臉: 6 筆
數字人/口型同步: 4 筆
(empty): 119 筆
```

mazz68的賣場自訂分類：
```

```

對應關係需重新拉齊，並補上 119 筆 (empty) 的分類歸屬。

### Phase 2 — 建立 Config

`docs/config/custom-categories.yaml` 存放分類名稱 → 關鍵字 → 各店 ID 對照。

```yaml
categories:
  換臉換衣:
    keywords: [換臉, 換衣, 換裝, faceswap, 換頭]
    stores:
      台南店: { category_id: 274800931 }
      高雄店: { category_id: null }  # 待補充
  語音克隆/數字人:
    keywords: [語音, 克隆, 數字人, 配音, 變聲]
    stores:
      台南店: { category_id: 274800921 }
      高雄店: { category_id: null }
  圖片生成/編輯:
    keywords: [圖片, 圖像, 照片, 繪圖, 修圖, 改圖]
    stores:
      台南店: { category_id: 274800865 }
      高雄店: { category_id: null }
  影片生成:
    keywords: [影片, 視頻, video, 生成]
    stores:
      台南店: { category_id: 274800945 }
      高雄店: { category_id: null }
  音樂生成:
    keywords: [音樂, 音頻, 歌曲, 音訊, 聲音]
    stores:
      台南店: { category_id: 274800956 }
      高雄店: { category_id: null }
  本地端大模型:
    keywords: [整合包, 一鍵部署, 本地, ollama, 離線]
    stores:
      台南店: { category_id: 274800967 }
      高雄店: { category_id: null }
  其他:
    keywords: []
    stores:
      台南店: { category_id: 268495060 }
      高雄店: { category_id: null }
```

Config 與 JSON 的關係：
- JSON 的 `category` 欄位存分類名稱（如 `"換臉換衣"`）
- Config 將分類名稱對應到：搜尋關鍵字 + 各店自訂分類 ID
- 新增/刪除分類時：改 config（ID 跟著走），補 JSON 的 `category` 值
- 程式碼讀 config，不需要因分類變動而改程式

### Phase 3 — 標題關鍵字

新商品上架時，標題內要包含 config 中對應的關鍵字。目的是讓 Phase 4 可以靠搜尋找到。

舊商品如果標題已有自然關鍵字（如「AI**換臉**工具」），搜尋時就會出現，不需要改標題。缺關鍵字的再補。

### Phase 4 — 上架後批次指派模組

在 batch-upload.html 加一個新功能「指派自訂分類」，或寫一支獨立腳本。

流程：
1. 讀取 product-catalog-tw.json → 依 `category` 分組
2. 讀取 custom-categories.yaml → 取得關鍵字 + 各店 category ID
3. 針對每個分類：
   - 開 seller.shopee.tw/portal/decoration/category/{categoryId}
   - 利用「新增商品」功能搜尋關鍵字
   - 批次選取 → 加入分類

實作方式有二：

| 方式 | 原理 | 優點 | 缺點 |
|------|------|------|------|
| A. 搜尋關鍵字批次加 | 複製人工操作流程：進分類 → 搜尋標題關鍵字 → 全選 → 新增 | 跟蝦皮 UI 一致，穩定 | 依賴關鍵字精準度 |
| B. 直接呼叫 API | 觀察「新增商品」的 XHR，模擬 POST 請求 | 快，不受 UI 變動影響 | 需要逆向 API，可能被風控 |

建議先實作方式 A（穩定），若速度不夠再進階到 B。

### 多店舖相容

config 已設計 `stores` 層級：
- 同一分類名稱，不同店有不同的 `category_id`
- 批次上傳時指定目標店鋪 → 自動查表取對應 ID
- 高雄店上架 → 用高雄店的 ID

## 實作順序

```
Phase 1: 拉齊 JSON category 值（需要你確認 119 筆 (empty) 的分類）
Phase 2: 建立 custom-categories.yaml
Phase 3: 補標題關鍵字（需要你決定）
Phase 4: 實作 batch-assign-category.js（批次指派模組）
```

Phase 1-3 可先做，Phase 4 等前三個穩定再實作。
