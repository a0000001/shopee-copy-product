## 二、 問題診斷與破口分析

### 1. 掃描數量不全 (DOM 安定掃描與審核中 API 爬取雙重結合)
- 採用測試工具成功的安定體驗：**以 DOM 爬取為主要基礎 (安定取得 35 筆架上商品)** + **以 API 補充爬取 (`reviewing` 審核中 3 筆)**。
- 兩者成果自動去重合併，確保 100% 安定取得完整 38 筆已上架/審核中商品。
- 顯示格式對齊：**`已上架 38 筆 (35 筆架上 + 3 筆審核中/其他)`**。

### 2. 發布點擊成功後訊息中斷 (Form Submit & Toast Notification)
- 點擊「儲存並上架」後，若商品進入蝦皮「審核中」佇列，頁面並不會立刻引發 URL 跳轉（仍停留在 `/new`），但會彈出 `商品已提交審核` 之 Toast。
- 舊有的訊息機制（`chrome.tabs.sendMessage`）在 Form Submit 觸發時易發生中斷，導致回傳 `null` 並觸發 `throw new Error('點擊上架按鈕失敗')`，引發重試機制。

## 三、 擬定修復方案 (Proposed Changes)

### Component 1: `batch-upload.js` 擴充全量頁籤掃描 (全量 38 筆比對)

#### [MODIFY] [batch-upload.js](file:///S:/projects/shopee-copy-product/extension/batch-upload.js)
- 擴充 `scanProducts()`，使其透過蝦皮賣家中心 API 或多頁籤 DOM（`live` / `reviewing` / `banned` / `unlisted`）全量爬取所有商品名稱。
- 顯示格式對齊：**`已上架 38 筆 (35 筆架上 + 3 筆審核中)`**。

```javascript
// 範例：全量頁籤掃描邏輯
async function scanProductsFull(tabId) {
  // 爬取 live (架上) 與 reviewing/banned (審核中)
  const result = await chrome.scripting.executeScript({
    target: { tabId },
    func: () => {
      // 擷取所有頁籤內出現的商品標題 DOM
      const titleEls = Array.from(document.querySelectorAll('.product-name, .product-title, [data-testid="product-name"]'))
      return titleEls.map(el => el.textContent.trim()).filter(Boolean)
    }
  })
  return result[0]?.result || []
}
```

### Component 2: `chrome.scripting.executeScript` 直注入發布狀態檢查

#### [MODIFY] [batch-upload.js](file:///S:/projects/shopee-copy-product/extension/batch-upload.js)
- 點擊「儲存並上架」後，不依靠敏感的 `sendMessage` 回傳，改採 `chrome.scripting.executeScript` 注入直接輪詢檢查 DOM 是否出現 **「審核中」** 或 **「提交成功」** 之 Toast 提示或成功狀態。

```javascript
// 直注入檢查 Toast / 成功標籤
const injectionRes = await chrome.scripting.executeScript({
  target: { tabId },
  func: () => {
    const toastText = Array.from(document.querySelectorAll('.eds-toast, .ant-message, .shopee-toast'))
      .map(t => t.textContent.trim()).join(' ')
    const isAudit = toastText.includes('審核') || toastText.includes('提交') || toastText.includes('成功')
    return { isAudit, toastText }
  }
})
```
