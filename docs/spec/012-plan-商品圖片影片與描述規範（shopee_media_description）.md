# 商品圖片、影片與描述規範

> Phase 1 盤點完成後，進入 Phase 2 補內容的規格依據。

---

## 一、圖片規範

### #image-specs

| 項目 | 規格 |
|------|------|
| 檔案格式 | `.jpg`（JPG），不接受 PNG / WebP / GIF |
| 檔案大小 | 每張 < 2MB |
| 數量下限 | 最少 5 張，建議 9 張（蝦皮支援上限） |
| 命名規則 | `[商品名稱]_[序號].jpg`，序號 001 起 |
| 尺寸建議 | 至少 1000×1000 px（蝦皮最佳顯示） |
| 色彩空間 | sRGB |

參考：https://seller.shopee.tw/edu/article/17316

### #image-types

| 張數 | 類型 | 說明 |
|------|------|------|
| 001 | 主圖 | 產品外觀/封面，白底或簡潔背景 |
| 002 | 功能圖 | 主要功能展示（UI 截圖或合成圖） |
| 003 | 應用場景 | 實際使用情境展示 |
| 004 | 規格說明 | 硬體需求、版本資訊 |
| 005~009 | 其他展示 | 特色功能、比較圖、效果對比等 |

### #image-source-priority

圖片來源優先順序（由優至劣）：

| 優先級 | 來源 | 說明 |
|--------|------|------|
| 1 | 實際產品截圖 | 最真實，顧客信任度最高 |
| 2 | 實際應用照片 | 產品使用情境實拍 |
| 3 | 模板合成圖 | 固定模板套上產品資訊（主圖、規格圖） |
| 4 | AI 生成示意圖 | 暫緩實施，見下方注意事項 |

> **Current Status**：Phase 2 目前僅處理 PNG→JPG 轉檔，AI 生成示意圖暫緩。

### #image-generation-caveats（待辦注意事項）

當未來啟用 AI 生成示意圖時，必須遵守以下規範：

| # | 注意事項 | 說明 |
|---|---------|------|
| 1 | **禁止誤導** | AI 生成圖不可誇大或虛構產品不存在的功能。示意圖必須加註「僅供示意」 |
| 2 | **真實優先** | 有實際 UI 截圖可用的產品，優先使用截圖而非 AI 生成 |
| 3 | **品質門檻** | AI 生成圖必須通過視覺檢查：無變形、無多餘手指/文字、色彩正常 |
| 4 | **版權迴避** | 不可生成包含其他品牌 Logo、角色、商標的圖片 |
| 5 | **風格一致** | 同一商品的多張 AI 圖必須風格統一（色調、構圖、字體） |
| 6 | **不生成 UI** | 不可用 AI 生成假的產品操作介面（顧客會當作實際功能） |
| 7 | **NSFW 處理** | NSFW 商品的示意圖不可暴露，以功能說明圖代替 |
| 8 | **Cost & Time** | AI 生圖+人工篩選的成本不能高於直接截圖+後製 |

### #conversion

```bash
# PNG → JPG 轉換（使用 Pillow）
# 透明背景自動填白，quality=85，optimize=True
python -c "
from PIL import Image
img = Image.open('input.png').convert('RGBA')
bg = Image.new('RGB', img.size, (255, 255, 255))
bg.paste(img, mask=img.split()[3])
bg.save('output.jpg', 'JPEG', quality=85, optimize=True)
"
```

**轉換腳本**：`docs/scripts/convert_png_to_jpg.bat`（批次掃描 94 個商品資料夾）

| 功能 | 說明 |
|------|------|
| 遞迴掃描 | 自動遍歷 `蝦皮商品圖稿\` 下所有子資料夾 |
| 透明處理 | RGBA PNG 以白色背景填充後轉換 |
| 略過機制 | 若 JPG 已存在則自動略過（可重複執行） |
| 轉換統計 | 輸出總數、轉換數、略過數、錯誤數 |

## 二、影片規範

### #video-specs

| 項目 | 規格 |
|------|------|
| 格式 | MP4（H.264 編碼） |
| 長度 | 10 ~ 60 秒 |
| 大小 | ≤ 30MB |
| 解析度 | ≥ 1×1（建議 1080×1080 或 1920×1080） |
| 編碼 | 不含 vp9 |
| 檔案位置 | 可直接放在商品資料夾根目錄 |

### #video-sources

| 來源 | 方法 | 適用場景 |
|------|------|---------|
| 現有素材 | 資料夾內已有 .mp4 | 直接使用 |
| 示範錄製 | 用 OBS / ScreenRec 錄製產品操作 | 有實際軟體可執行者 |
| 大陸網站爬取 | Bilibili / 抖音下載教學或展示影片 | 需要用戶提供 cookie/token |
| AI 生成 | Runway / Pika / Kling 等生成展示畫面 | 無實際軟體只有說明時 |

## 三、商品描述規範

### #description-structure

每個「商品描述.md」包含兩層內容：

#### 專屬部分（每商品不同）

```
## 產品介紹

[功能說明：3~5 句話，說明產品能做什麼、特色、使用場景]

## 使用方式

[使用方式：簡潔步驟式說明，或貼心提示]

## 電腦規格建議

- 最低需求：[由 AI 根據產品類型判斷]
- 建議配備：[由 AI 根據產品類型判斷]

## Tags

[#Tag1 #Tag2 #Tag3 ...]
```

#### 共用部分（全賣場統一）

```
## 📦 購買須知（重要！請務必閱讀）

### 硬體相容性
- 本商品為軟體工具，請確認您的電腦符合需求規格再下單
- 建議先使用硬體檢測工具確認相容性

### 載點說明
- 下單後透過聊聊訊息提供載點
- 載點時效：收到後請儘速下載
- 如連結失效請聯繫補發

### 退換貨政策
- 虛擬商品售出後恕不退換
- 若載點無法使用，提供換貨處理
- 請於收貨後 7 日內確認商品正常

### 貨到付款
- 支援 7-11 / 全家取貨付款
- 虛擬商品出貨方式選擇「賣家宅配」或「無需物流」
- 請於訂單備註提供 Email 以利寄送載點
```

### #notion-constraint

> **CRITICAL**：AI 生成商品描述時，不得包含任何 AI 前後文說明（如「這是由AI生成的內容...」「根據您的需求...」等）。描述必須以商品本位直接撰寫，彷彿是人類賣家撰寫的商品文案。

## 四、檔案結構標準

### #final-directory-standard

```
[商品名稱]/
├── images/
│   ├── [商品名稱]_001.jpg   # 主圖
│   ├── [商品名稱]_002.jpg   # 功能圖
│   ├── [商品名稱]_003.jpg   # 應用場景
│   ├── [商品名稱]_004.jpg   # 規格說明
│   ├── [商品名稱]_005.jpg   # 其他
│   └── ...                  # 最多到 009
├── videos/
│   └── [商品名稱]_demo.mp4  # 展示影片（選配）
├── 商品描述.md              # 商品文案
├── 載點.txt                 # 下載連結（選配）
└── listing.json             # 上架用結構化資料（自動生成）
```

## 五、mazz68 商品描述爬取策略

### #crawl-strategy

1. **目標**：爬取 mazz68 蝦皮賣場各商品的「商品描述」作為文案參考
2. **工具**：Python `requests` + `BeautifulSoup`（或用 Selenium 處理 JavaScript 渲染）
3. **所需資料**：賣家登入後的 session cookie（需用戶提供）
4. **配對方式**：爬回的商品標題 → 比對 `product-catalog.json` 的 `product_name.zh-TW` → 合併至對應資料夾

### #crawl-output

爬取結果格式（JSON）：

```json
{
  "shopee_title": "蝦皮上的完整標題",
  "shopee_description": "原始商品描述 HTML/Markdown",
  "matched_product": "product-catalog.json 對應名稱",
  "status": "matched / unmatched"
}
```
