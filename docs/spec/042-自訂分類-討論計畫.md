# 自訂分類 — Phase 2 討論與實作計畫

## 問題回顧

1. 自訂分類（賣場佈置 → 我的賣場分類）在「新增商品」頁面沒有對應欄位，上架時無法設定
2. `product-catalog-tw.json` 已有 `category` 欄位，但只是內部標記，批次上架也沒地方填
3. 原 mazz68 賣場的分類跟我們自己的分類系統不一致
4. 跨店（台南/高雄）的自訂分類 ID 不同, 這點不管了。

## 最終決策

### 放棄沿用 mazz68 分類

完全放棄 mazz68 的分類系統，改用我們自己的分類。

**為何現在改還來得及**：user 剛開始在蝦皮建立賣場，尚未大量使用 mazz68 的分類，所以全部重分沒有包袱。

**已廢除的舊分類（mazz68）**：
- `實用工具` → 按實際功能分散到各類（音樂生成/影片生成/語音配音等），無法判斷的放「其他」
- `影片處理` → 併入 `影片生成`
- `虛擬換衣/換臉` → 併入 `換臉換衣`
- `數字人/口型同步` → 併入 `語音/配音/變聲`
- `圖片處理` → 併入 `圖片生成/編輯`

**注意**：舊分類 `實用工具` 原本有 18 筆，經逐一檢視描述後分配到各類，未有無法決定的情形，故無需額外丟入「其他」。

## 核心機制：標題嵌入 Hashtag

### 標題格式

```
{原本商品名稱} #{category}#{tag1}#{tag2}#{tag3}
```

範例：
```
MOSS-TTS,介面API,語音合成,8G顯存可用,聲音複製,TTS,支援超長文字,支援50系顯示卡,支援老顯示卡 #語音/配音/變聲#語音合成#TTS#語音#Voice
```

### 標題長度限制

蝦皮規定標題最長 60 字元。
若嵌入 hashtag 後超過 60 字元，**以最後一個 `#` 為分隔點切斷**，包含該 `#` 及之後所有文字都刪除。

實作邏輯（`seller-fill.js` 的 `fillAll`）：

```js
function buildTitle(productName, category, tags) {
  const tagStr = [category, ...tags].filter(Boolean).join('#')
  const hashtag = tagStr ? ' #' + tagStr : ''
  const maxLen = 60

  if ((productName + hashtag).length <= maxLen) {
    return productName + hashtag
  }

  // 超過長度，逐步捨棄 hashtag
  const tagParts = [category, ...tags]
  let remaining = []
  let currentLen = productName.length + 1  // +1 for space before #
  
  for (const part of tagParts.reverse()) {
    const wouldAdd = 1 + part.length  // # + text
    if (currentLen + wouldAdd <= maxLen) {
      remaining.unshift(part)
      currentLen += wouldAdd
    } else {
      break
    }
  }
  
  if (remaining.length === 0) {
    // 連一個 tag 都塞不下，只回產品名稱（截斷到 maxLen）
    return productName.substring(0, maxLen).trim()
  }

  return productName + ' #' + remaining.join('#')
}
```

### 比對去重時要忽略 Hashtag

`batch-upload.js` 的 `scanProducts()` 會掃描已上架商品的標題來比對目錄，避免重複上架。
因為實際 Shopee 標題已包含 hashtag，但 JSON 目錄的 `ps_product_name` 不包含 hashtag，直接比對會失敗。

修正：掃描時從 Shopee 標題中**移除 `#` 及之後的所有文字**再比對。

```js
function stripHashtag(title) {
  const idx = title.indexOf(' #')
  return idx >= 0 ? title.substring(0, idx).trim() : title.trim()
}
```

影響位置（`batch-upload.js`）：
- Line 84：`n = stripHashtag(item.name)`
- Line 110：`n = stripHashtag(p.name)`

## 商品描述嵌入 Hashtag + Computer Specs

### 描述結尾格式

```
{原本商品描述}

---

⚙️ 建議配備
• 最低 VRAM：{vram_min} GB（{vram_rec ? `建議 ${vram_rec} GB` : ''}）
• 最低 RAM：{ram_min} GB（{ram_rec ? `建議 ${ram_rec} GB` : ''}）
• 硬碟空間：{disk} GB
• CPU：{cpu}
• 電源：{power}

#{category}#{tag1}#{tag2}#{tag3}
```

最後一行放 hashtag 是為了讓蝦皮的「我的賣場分類 → 新增商品 → 搜尋」功能可以搜到這些關鍵字。Shopee 的商品搜尋涵蓋商品描述內容。

### computer_specs 補充方式

`computer_specs` 欄位的設計與補充規則，定義於 `docs/data/從mazz68收錄並拆分商品描述.md`：

1. **第一步**：到 `S:\projects\ai-hardware-checker\models.json` 查該模型
2. **第二步**：若有條目，直接引用 `vram_min/vram_rec/ram_min/ram_rec/disk`
3. **若無條目**：上 Hugging Face 或 GitHub 找官方推論硬體需求，或參照同類型模型推斷
4. **第三步**：確實填寫，不得寫「雲端服務」「無需特定」這類廢話

欄位結構：
```json
{
  "vram_min": 8,
  "vram_rec": 12,
  "ram_min": 16,
  "ram_rec": 32,
  "disk": 10,
  "cpu": "Intel i5-10400 / AMD Ryzen 5 3600 或以上",
  "power": "無特別要求"
}
```

目前已有資料：`product-catalog-tw.json` 中多數商品 `computer_specs` 只填了 `vram_min`，其餘為 0 或空字串，需要逐步補齊。

## 自訂分類 ID 對應（Config）

不需要複雜的 config 了。因為：
- 自訂分類只在「賣場佈置 → 我的賣場分類」手動設定
- 上架時根本不能填自訂分類
- Hashtag 機制只是讓商品標題/描述含關鍵字，方便後續在「我的賣場分類」中用搜尋功能批次加入

所以 ID 只需要在**手動去賣場分類設定時**查一下就好，不需要寫死在程式碼裡。

記錄方式：直接在 `docs/data/自訂分類ID對照.md` 隨手記


## 標題長度實測確認

```
MOSS-TTS,介面API,語音合成,8G顯存可用,聲音複製,TTS,支援超長文字,支援50系顯示卡,支援老顯示卡
```

上述標題（不含 hashtag）已經 54 字元 + hashtag ` #語音/配音/變聲#語音合成#TTS#語音#Voice` = 32 字元 → 總長 86 字元，超過 60。

### 實際長度分布統計（207 筆）

| 條件 | 可塞入數量 | 佔比 |
|------|-----------|------|
| 可塞下 `#{category}` | 196 | 94.7% |
| 可塞下 `#{category}+1tag` | 193 | 93.2% |
| 可塞下 `#{category}+2tags` | 184 | 88.9% |
| 可塞下 `#{category}+3tags` | 167 | 80.7% |
| 連 `#{category}` 都塞不下 | 11 | 5.3% |

結論：大部分商品不只可以塞 category，還可以塞 1-3 個 tag。當前實作只用了 category 是未完成的 bug。

## 修正後實作規格

### seller-fill.js

```js
function buildTitle(productName, category, tags) {
  // 依序嘗試塞入 #{category}#{tag1}#{tag2}#{tag3}
  // 超過 60 字元時從最後一個 tag 開始放棄
  // category 永遠優先保留
  // 連 category 都塞不下才截斷商品名稱
}
```

```js
function buildDescription(product, config) {
  // 1. 保留原始商品描述
  // 2. 有 computer_specs 資料 → 輸出建議配備區塊
  // 3. 無論有無 computer_specs，只要有 config.hardware_check_url → 輸出硬體檢測連結
  // 4. 輸出 #{category}#{tag1}#{tag2}#{tag3}
  // 5. 確保 hashtag 在任何情況下都有輸出（就算沒 category 也至少有 #{其他} 兜底）
}
```
