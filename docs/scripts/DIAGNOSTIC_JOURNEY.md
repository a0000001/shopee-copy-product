# `__remo__/` — 診斷工具集

本目錄收錄的商品頁診斷腳本，用於定位蝦皮商品頁圖片擷取不足的根因。

---

## 檔案一覽

### `_000_PROVEN_rootcause_carousel_virtual_rendering.js` ⭐

**找到根因的關鍵腳本。** 請勿刪除。

在蝦皮商品頁 Console 執行後，自動完成 4 階段對照實驗：

| Phase | 操作 | 結果 |
|---|---|---|
| 1 | 立即拍照 | 5 個縮圖 |
| 2 | 點擊右箭頭 | 仍是 5 個 |
| 3 | 點擊所有縮圖 | **暴增到 9 個** |
| 4 | 連續滾動 | 維持 9 個 |

**確立根因**：蝦皮 carousel 使用 virtual rendering，初始只 render 可見的 5 個縮圖，其餘 4 個的 DOM 節點**根本不存在**。點擊縮圖會觸發 React state 更新，才將完整 9 張 render 到 DOM 中。

---

### `_001_diagnostic_dom_timing.js`

第一輪診斷：確認問題是否為「DOM 尚未載入完整就提取」。

執行方式：立即拍照 → 等 8 秒 → 再拍 → 比對。

**結論**：t0 = 5 張，t8 = 5 張，數量不變。**不是 timing 問題。** 排除了「等 React 渲染」這個方向。

---

### `_002_diagnostic_dom_click.js`

第二輪診斷：探索輪播互動能否觸發更多圖片載入。

自動尋找並點擊右箭頭按鈕、點擊所有縮圖，再比對。

**結論**：右箭頭無效；點擊縮圖才有效。為 `_000` 的全面測試鋪路。

---

## 診斷歷程

```
問題：popup 只顯示 5 張圖，但商品有 9 張
  │
  ├─ 假設 1：DOM 還沒載完（timing）
  │   └─ _001 測試 → ❌ 等 8 秒也沒增加
  │
  ├─ 假設 2：輪播需要 scroll 才會載入
  │   └─ _002 測試 → ❌ 右箭頭無效，但點縮圖有效
  │
  └─ 假設 3：virtual rendering，縮圖根本不在 DOM 裡
      └─ _000 測試 → ✅ 點縮圖後從 5 張變 9 張
                      ✅ 14 個 mdCA_C 容器（含主圖區、popup）
```

根因：**蝦皮 carousel virtual rendering**（不是 timing，不是 scroll，是 React state 沒觸發）。
解法：提取圖片前先程式化點擊所有 `.mdCA_C` 縮圖，觸發完整渲染後再提取。
